# venvy — менеджер виртуальных окружений

Кратко

    venvy создаёт и управляет локальными Python virtualenv в каталоге пользователя.
    Команды вызываются просто: venvy, venvy use <имя>, venvy list и т.д.

Установка

    Скопируйте исполняемый файл в ~/ .local/bin и убедитесь, что этот каталог в PATH: mkdir -p ~/.local/bin cp /путь/к/venvy ~/.local/bin/ chmod +x ~/.local/bin/venvy

    Добавьте в ~/.bashrc или ~/.zshrc (если ещё не добавлено):
    export PATH="$HOME/.local/bin:$PATH"

    Откройте новый терминал или выполните:
    source ~/.bashrc   # или source ~/.zshrc

Использование

    venvy
        Создаёт окружение default (если оно отсутствует) и открывает новую bash‑сессию с активированным окружением.
        Пример: venvy (venvy:default) user@host:~$

    venvy use <имя>
        Создаёт окружение с именем <имя> (если нет) и открывает shell в нём.
        Пример: venvy use myenv (venvy:myenv) user@host:~$

    venvy list
        Показывает список всех окружений (содержимое каталога VENVY_HOME).

    venvy remove <имя>
        Удаляет окружение <имя>. Внимание: удаляет каталог без подтверждения.

    venvy run <имя> <скрипт> [args...]
        Создаёт (при необходимости) окружение <имя> и запускает указанный скрипт в нём: venvy run myenv path/to/script.py arg1 arg2

Переменные окружения

    VENVY_HOME — каталог, в котором хранятся окружения. По умолчанию: VENVY_HOME="${HOME}/.venvy"

Расположение окружений

    Окружения создаются как подкаталоги в $VENVY_HOME, например: ~/.venvy/default ~/.venvy/myenv

Замечания и рекомендации

    venvy выполняет exec /bin/bash --rcfile ... для запуска новой интерактивной сессии с активированным окружением; при выходе из этой сессии вы вернётесь в предыдущий shell.
    Не запускайте venvy под sudo — окружения будут созданы в /root и будут недоступны обычному пользователю.
    Для удаления окружений используется rm -rf — будьте осторожны.
    Если вы используете zsh, учтите, что venvy запускает bash для корректного поведения активации виртуального окружения.

Примеры

    Создать и войти в default:
    venvy

    Создать и войти в окружение test:
    venvy use test

    Запустить скрипт в окружении myenv:
    venvy run myenv scripts/my_script.py --option
